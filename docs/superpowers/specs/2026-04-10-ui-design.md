# UI 设计规格 — 黑夜君临遗物助手

## 概述

使用纯 PySide6 原生控件构建桌面 GUI，实现 build 管理、通用词条管理、黑名单管理、设置和词条库加载等平台无关功能。

## 技术选型

- 框架：PySide6 原生控件（不使用 FluentWidgets）
- 导航：QTabWidget 标签页切换
- 持久化：JSON 文件
- 词条库：只读 txt 文件

## 整体架构

```
MainWindow (QMainWindow)
└── QTabWidget (5个标签页)
    ├── HomePage        — 占位，显示项目信息
    ├── BuildPage       — 角色 build 管理
    ├── CommonPage      — 通用词条组管理
    ├── BlacklistPage   — 黑名单组管理
    └── SettingsPage    — 设置
```

代码组织：
```
src/nra/
├── main.py
├── ui/
│   ├── main_window.py       — MainWindow + QTabWidget
│   ├── pages/
│   │   ├── home_page.py
│   │   ├── build_page.py
│   │   ├── common_page.py
│   │   ├── blacklist_page.py
│   │   └── settings_page.py
│   └── widgets/
│       └── affix_editor.py  — 搜索+添加的词条编辑组件（复用）
├── models/
│   ├── preset_manager.py    — 预设 CRUD 与持久化
│   └── vocabulary.py        — 词条库加载与搜索
```

## 页面设计

### 主页 (HomePage)

占位页面，显示项目名称和版本号。后续接自动化流程的状态和日志输出。

### Build 管理 (BuildPage)

左右分栏布局（QSplitter）：

**左侧 — build 列表：**
- QListWidget 显示所有 build
- 下方按钮：新建、删除
- 双击 build 名称可重命名

**右侧 — build 编辑区：**
- build 名称（QLineEdit，编辑后自动保存）
- "包含通用词条" checkbox + 通用组多选（勾选引用哪些通用组）
- "引用黑名单" + 黑名单组多选（勾选引用哪些黑名单组）
- min_matches（QSpinBox，命中几条算合格，默认 2）
- QTabWidget 区分普通/深夜白名单编辑：
  - 普通白名单：AffixEditor 组件（搜索+添加，词条源 = normal.txt + normal_special.txt）
  - 深夜白名单：AffixEditor 组件（词条源 = deepnight_pos.txt）

### 通用管理 (CommonPage)

左右分栏布局：

**左侧 — 通用组列表：**
- QListWidget 显示所有通用组
- 下方按钮：新建、删除

**右侧 — 通用组编辑区：**
- 组名称（QLineEdit）
- QTabWidget 区分普通/深夜：
  - 普通词条：AffixEditor（词条源 = normal.txt + normal_special.txt）
  - 深夜词条：AffixEditor（词条源 = deepnight_pos.txt）

### 黑名单管理 (BlacklistPage)

左右分栏布局：

**左侧 — 黑名单组列表：**
- QListWidget
- 下方按钮：新建、删除

**右侧 — 黑名单组编辑区：**
- 组名称（QLineEdit）
- AffixEditor（词条源 = deepnight_neg.txt）
- 黑名单只有深夜负面词条，不分普通/深夜

### 设置 (SettingsPage)

- 词条库路径配置（QLineEdit + 浏览按钮）
- 数据存储路径配置
- 预留快捷键、Steam 路径等后续配置项

## 复用组件

### AffixEditor

搜索+添加的词条编辑器，在 build/通用/黑名单页面中复用。

结构：
- 顶部：QLineEdit 搜索框，输入文字实时过滤词条库
- 中部：QListWidget 搜索结果列表（点击或回车添加）
- 底部：QListWidget 已选词条列表（支持删除，选中后按 Delete 或右键删除）

输入：词条库来源（由调用方指定加载哪些 txt 文件）
输出：已选词条列表（list[str]）

## 数据模型

### 词条库 (只读资源)

文件位于 `data/` 目录：
- `normal.txt` — 普通模式词条
- `normal_special.txt` — 普通模式特殊词条
- `deepnight_pos.txt` — 深夜模式正面词条
- `deepnight_neg.txt` — 深夜模式负面词条

格式：每行一条词条，支持 `→` 分隔格式（取右侧）。

### VocabularyLoader

- `load(files: list[str]) -> list[str]`：加载指定文件的词条
- `search(query: str, vocabulary: list[str]) -> list[str]`：模糊搜索词条

### 预设数据 (用户数据)

文件：`data/presets.json`

```json
{
  "version": "1.0",
  "common_groups": [
    {
      "id": "uuid-string",
      "name": "属性通用",
      "normal_affixes": ["生命力+3", "强韧度+3"],
      "deepnight_affixes": ["提升血量上限"]
    }
  ],
  "blacklist_groups": [
    {
      "id": "uuid-string",
      "name": "致命负面",
      "affixes": ["持续减少血量", "降低物理减伤率"]
    }
  ],
  "builds": [
    {
      "id": "uuid-string",
      "name": "火法师",
      "include_common": true,
      "common_group_ids": ["uuid-string"],
      "blacklist_group_ids": ["uuid-string"],
      "min_matches": 2,
      "normal_whitelist": ["提升火属性攻击力+3"],
      "deepnight_whitelist": ["提升火属性攻击力+3"]
    }
  ]
}
```

### PresetManager

- `load()` / `save()`：读写 presets.json
- Build CRUD：`create_build()`, `update_build()`, `delete_build()`
- Common group CRUD：`create_common_group()`, `update_common_group()`, `delete_common_group()`
- Blacklist group CRUD：`create_blacklist_group()`, `update_blacklist_group()`, `delete_blacklist_group()`
- 文件不存在时自动创建空结构

### 设置数据

文件：`data/settings.json`

```json
{
  "vocab_dir": "data",
  "data_dir": "data"
}
```

## 信号与数据流

- 各页面通过 PresetManager 共享数据
- 编辑操作直接调用 PresetManager 方法，PresetManager 自动保存
- Build 页面引用通用/黑名单组时，通过 id 关联
- 删除通用/黑名单组时，自动清理 build 中的引用
